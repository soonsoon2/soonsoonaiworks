# Repository Principles

이 저장소는 프로젝트 공통 AI 작업 방식, 작업환경 운영 방법, 반복 재사용 리소스만 관리한다.

## Rules
- 먼저 `README.md`를 읽고, 기존 섹션을 작게 수정한다.
- 프로젝트별 로그, 회의 메모, 실험 기록, 임시 결정, 비밀 정보는 추가하지 않는다.
- 한 프로젝트에만 필요한 규칙은 해당 프로젝트의 지시 문서에 둔다.
- 여러 프로젝트에서 반복 확인된 안정적인 방법만 이 저장소로 승격한다.
- 공용 리소스는 `resources/` 아래에 두고, 실행 방법과 유지 기준을 해당 README에 짧게 적는다.
- 새 파일은 `README.md`가 읽기 어려울 만큼 길어졌을 때만 만든다.
- 변경 후 문서 구조와 링크를 확인하고, 무엇을 바꿨는지 짧게 보고한다.
